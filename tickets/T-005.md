# T-005 — 알림 채널 라우팅 + 전송

**상태**: 🟢 DONE  
**담당**: Codex  
**의존**: T-004

---

## 목표

출발 알림을 실제 채널로 전송한다.
MVP는 Discord webhook과 Telegram Bot을 지원하고, 설정으로 단일/복수 채널을 선택할 수 있게 한다.

---

## Acceptance Criteria

- [x] `core/notifier.py`의 공용 전송 함수 구현
- [x] `settings.yaml`의 `notification.enabled_channels`로 활성 채널 선택 가능
- [x] `discord`, `telegram`, `둘 다` 구성 지원
- [x] Discord는 `DISCORD_WEBHOOK_URL` 환경변수로 전송
- [x] Telegram은 `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID` 환경변수로 전송
- [x] 활성 채널에 따라 `main.py` 환경변수 검증이 조건부로 동작
- [x] 공통 알림 메시지 포맷 구현
- [x] 채널 하나가 실패해도 다른 채널 전송은 계속 시도
- [x] 단위 테스트: `tests/test_notifier.py` 작성

---

## 구현 상세

### settings.yaml 예시
```yaml
notification:
  enabled_channels:
    - discord
    - telegram
```

### 채널 정책
- 활성 채널은 설정 리스트 기준으로 결정
- 미지원 채널 값은 무시하지 말고 명확히 에러 처리
- 개별 채널 실패는 결과로 남기고 다른 채널은 계속 시도

### 알림 메시지
기존 텔레그램 포맷을 기본으로 유지하되, Discord와 Telegram 모두에서 읽기 쉬운 텍스트 메시지로 전송
