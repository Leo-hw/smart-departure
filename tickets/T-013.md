# T-013 — public 안전화: 로그/캐시 민감정보 제거

**상태**: 🟡 READY  
**담당**: Codex  
**의존**: T-012  
**우선순위**: 🔥 긴급 (public repo에서 개인 일정 데이터 노출 중)

---

## 배경

repo를 public으로 전환한 결과, 개인 캘린더 데이터가 공개 노출됨:
- **Actions 로그(누구나 열람)**: `main.py`가 매 실행마다 일정명(`summary`)과 `event_id`를 출력
- **Actions 캐시(공개 repo는 PR 워크플로우로 접근 가능)**: `.runtime/schedule_today.json`에 일정명 + **장소(전체 주소)** 적재, `sent_alerts.json`에 `event_id` 원문

Discord 메시지(본인 비공개 채널)는 누출 아님 → **로그와 캐시만** 안전화한다.

> 전제: 본 작업 중에는 누출 차단을 위해 repo를 임시 private로 두거나 워크플로우를 비활성화한다. 마스킹 완료 + 과거분 purge 후 다시 public.

---

## Acceptance Criteria

### A. 로그 마스킹 (모든 stdout/stderr)
- [ ] `main.py` plan 로그(L80~)에서 `summary` 제거, `event_id`는 짧은 해시(예: sha256 앞 8자)로만
- [ ] due 로그(L90~)의 `event_id` → 해시
- [ ] delivery failed 로그(L102~)의 `event_id` → 해시, `error`는 일정정보/웹훅URL이 새지 않게 정제
- [ ] startup failed / `send_failure_notification` 경로에서 예외 메시지에 summary/location/주소가 안 들어가게
- [ ] 유지해도 되는 것: 카운트(plans/due/attempted 수), `classification`, `alert_type`, provider, 추정여부
- [ ] 시각 출력은 유지 가능(저민감) — 단 summary/location과 절대 함께 묶지 말 것
- [ ] ⚠️ **Discord 메시지에는 `summary`/`location` 그대로 유지** — 마스킹은 GHA 로그/캐시 한정. 알림 본문은 본인 비공개 채널이라 일정명·장소가 있어야 쓸모 있음. notifier의 메시지 포맷은 건드리지 말 것.

### B. 캐시 민감정보 제거
- [ ] GHA 캐시 경로를 `.runtime/` 전체 → **`.runtime/sent_alerts.json`만** (schedule_today.json 캐시 제외)
- [ ] `schedule_today.json`은 런타임 한정 파일로만 유지 (ttl=0이라 재사용도 안 하므로 캐시 불필요)
- [ ] `core/dedup.py`의 dedup 키와 `_build_record`의 `event_id`를 **해시로 저장** (원문 event_id 미저장)
- [ ] dedup 동작(중복 방지, catch-up)은 해시 기반으로도 동일하게 유지 — 회귀 없음
- [ ] `event_start`는 만료 판정용으로 유지 가능(저민감: 시점만, 내용/장소 없음)

### C. 과거 노출분 purge
- [ ] 과거 workflow runs 전부 삭제
- [ ] 과거 Actions caches 전부 삭제
```bash
gh run list -R Leo-hw/smart-departure --limit 1000 --json databaseId -q '.[].databaseId' \
  | xargs -I{} gh api -X DELETE /repos/Leo-hw/smart-departure/actions/runs/{}
gh cache delete --all -R Leo-hw/smart-departure
```

### D. 검증
- [ ] 샘플 실행 로그에 일정명/주소/원문 event_id **없음** 확인
- [ ] 캐시 아티팩트(`sent_alerts.json`)에 원문 event_id/summary/location **없음** 확인
- [ ] 기존 테스트 + dedup 해시 회귀 테스트 통과
- [ ] (재공개 후) 미래 일정 1건으로 알림 정상 도착 + 로그 깨끗 동시 확인

## 구현 상세

### event_id 해시 헬퍼 (공용)
```python
import hashlib
def short_id(raw: str) -> str:
    return hashlib.sha256(raw.encode("utf-8")).hexdigest()[:8]
```
- 로그/캐시 모두 동일 해시 사용 → dedup 일관성 유지

### GHA 캐시 경로 변경
```yaml
- uses: actions/cache@v4
  with:
    path: .runtime/sent_alerts.json   # 디렉터리 전체 → 단일 파일
    key: runtime-${{ github.run_id }}
    restore-keys: |
      runtime-
```

### dedup 레코드 (변경 후)
```json
{
  "<hash8>_departure": {
    "status": "sent",
    "sent_at": "...",
    "event_start": "..."
  }
}
```
(원문 event_id / summary / location 미포함)
