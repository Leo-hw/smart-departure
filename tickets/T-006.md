# T-006 — 중복 알림 방지 (dedup)

**상태**: 🟢 DONE  
**담당**: Codex  
**의존**: T-005

## 목표

같은 일정에 대해 짧은 시간 안에 동일 알림이 반복 전송되지 않도록 TTL 기반 dedup을 적용한다.

---

## Acceptance Criteria

- [x] `.runtime/sent_alerts.json` 기반 dedup 구현
- [x] `settings.yaml`의 `schedule.dedup_ttl_minutes` 반영
- [x] 동일 `event_id`는 TTL 이내 재전송하지 않음
- [x] TTL이 지난 기록은 자동으로 만료 처리
- [x] 전송 성공한 일정만 dedup 기록으로 저장
- [x] 알림 전송 실패한 일정은 dedup에 기록하지 않음
- [x] `main.py` 실행 흐름에 dedup 필터 적용
- [x] 단위 테스트: `tests/test_dedup.py` 작성

---

## 구현 상세

### 저장 파일
```text
.runtime/sent_alerts.json
```

### 권장 저장 구조
```json
{
  "calendar-event-id": {
    "sent_at": "2026-04-16T14:15:00+09:00",
    "departure_time": "2026-04-16T14:15:00+09:00"
  }
}
```

### 동작 규칙
- dedup 키는 현재 MVP에서 `event_id`
- 활성 채널이 여러 개여도 이벤트 단위로 한 번만 dedup 처리
- 최소 한 채널이라도 성공 전송되면 해당 이벤트는 sent로 기록
